# SOLA v5.1.1

Hotfix release.

## What's fixed

The privacy acceptance banner shown to first-time learners had a literal placeholder leak in its heading: it rendered `Before you chat with {$a->product}` instead of `Before you chat with SOLA`.

Root cause was a Moodle `get_string` substitution-shape mismatch in `classes/hook_callbacks.php`. The lang string uses `{$a->product}` (object-property substitution), but the PHP call passed `branding::short_name()` as a bare string. When the substitution shapes do not match, Moodle leaves the placeholder unsubstituted and emits the literal placeholder text. All 46 language files already had the correct `{$a->product}` template; only the PHP call site was wrong.

Fixed by passing an object with a `product` property — the same shape that the `consent_body` substitution at the same call site already uses correctly.

Verified post-fix:
- `chat:consent_heading` renders as `Before you chat with SOLA`.
- No placeholder leakage in the rendered string.
- Validator corpus 30/30, jailbreak corpus 25/25.

## Backwards compatibility

Drop-in upgrade from v5.1.0. No setting changes, no schema changes, no behavioural changes outside the consent banner heading.

Plugin version `2026050501`, release `5.1.1`.
