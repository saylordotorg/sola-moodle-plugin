# SOLA v5.4.1

8 new Behat scenarios across two new feature files. Behat coverage went from 12 → 20 scenarios, 109 → 207 steps. All scenarios validated locally pre-tag in headless Chrome.

## What's new

### `tests/behat/quiz_flow.feature` — 4 scenarios

Pins the practice-quiz UI surface. The v5.3.x SOLA_NEXT-shaped JS-render bugs (`chip1||chip2`-style placeholders surfacing on screen) would surface here next time:

| Scenario | What it asserts |
|---|---|
| Quiz starter button opens the setup panel | Click `[data-starter="quiz"]` → `.aica-quiz-setup` is visible |
| Quiz setup cancel returns to the starters overlay | Click `.aica-quiz-setup__cancel` → starters re-appear, setup panel removed |
| Quiz setup Start button is wired and visible | `.aica-quiz-setup__start` and `.aica-quiz-setup__cancel` both render |
| Question count selector exposes 3 / 5 / 10 options | `.aica-quiz-setup__count-row` shows all three preset count labels |

The Start button → question card render isn't tested end-to-end here because the question-card render depends on the LLM-generated quiz JSON shape that's already covered by the v5.3.26 PHPUnit suite (`external_llm_calls_test::test_generate_quiz_*`). The Behat scenarios pin the UI button wiring, which is the surface that historically breaks.

### `tests/behat/drawer_interactions.feature` — 4 scenarios

Pins the four header buttons documented in `CLAUDE.md` (settings panel, reset, clear, help):

| Scenario | What it asserts |
|---|---|
| Settings panel opens from the gear icon | Click `.local-ai-course-assistant__btn-settings-panel` → settings panel visible |
| Reset (home) icon shows starters without clearing message history | Send a message → reset → starters re-appear AND the old message is still in scrollback (CLAUDE.md guarantee that v5.3.x regressed against) |
| Help button surfaces the in-drawer help panel | Click `.local-ai-course-assistant__btn-help` → `.aica-help-panel` visible |
| Two-cycle send-receive across a reset | Send → reset → send again — both messages survive in the messages container; tests conversation_manager state across mid-session resets |

The post-reset send-receive scenario is the most valuable of the four: it exercises the SSE pipeline twice with a reset between, which is the exact surface where v5.3.x had a regression where reset-and-resume left the conversation in a broken state.

## Why this is the right next investment

Of the three end-of-day open follow-ups, this is the highest-leverage one:

- **Test infrastructure for the next learner-facing bug.** Behat for the quiz flow and drawer interactions catches regressions in features SOLA's learners use every session.
- **No new infra cost.** Local Behat is already set up (v5.3.30), the stub provider is in place (v5.3.26), and the consent / intro user-pref bypass is documented (v5.3.29). New scenarios are effectively additive.
- **Validated locally before tag.** Per the v5.3.27 → v5.3.29 lesson, all 8 new scenarios were run end-to-end on the dev box in headless Chrome before this commit. CI exists as a check, not as the discovery mechanism.
