# SOLA v5.3.34

Mastery + flashcard SRS unit tests. 16 new PHPUnit tests covering `objective_manager` and `flashcard_manager` — until this release both core managers had no direct unit-test coverage despite running against every learner's review schedule and mastery promotion. **5 more `?:` fall-through bugs found and fixed** as a side effect of broadening the static-analysis test from v5.3.33 to also flag `?: self::CONST` patterns.

## What's new in tests

`tests/mastery_test.php` — 16 tests:

**objective_manager (10 tests)**

- CRUD lifecycle (create → get → update → delete).
- Delete cascades to `obj_att` attempt rows.
- `list_for_course` filters correctly.
- `import_batch` drops empty-title and whitespace-title rows.
- Per-course feature flag round-trip via `feature_flags::resolve`.
- Mastery formula: `not_started` for 0 attempts.
- Mastery formula: stays at `learning` below the `MIN_ATTEMPTS_FOR_MASTERY` (3) floor — no mastery on a lucky pair.
- Mastery formula: promotes to `mastered` after 5+ correct attempts (tested with relaxed threshold because the default 0.85 + WINDOW=8 + Bayesian prior 2/4 caps achievable score at ~0.79 — see footnote below).
- Mastery decay model: enabling decay lowers the score for old-timestamp attempts.
- `get_weak_objectives` surfaces lowest-scoring first.

**flashcard_manager (6 tests)**

- Per-course feature flag round-trip.
- `save_batch` drops empty-question and empty-answer rows.
- `save_batch` drops hallucinated objective IDs (the v4.0/M5 mastery boundary — LLM cannot inject a foreign objective tag onto a course's flashcards).
- SM-2 schedule: QUALITY_AGAIN resets repetitions to 0 and schedules next_review in ~10 minutes.
- SM-2 schedule: QUALITY_EASY grows the interval through reps 1 → 4d, 2 → 7d, 3+ → at least 7d with ease.
- `get_due` returns only cards whose `next_review` has passed.

## The 5 ?: bugs fixed in this release

The v5.3.33 static-analysis test only matched numeric / float literal defaults. Broadening the regex to also catch `?: self::CONSTANT` and `?: SomeClass::CONST` (inside `(int)` or `(float)` casts) surfaced 5 more sites:

| File | Setting | Default constant | What 0 was supposed to mean |
|------|---------|-------------------|----------------------------|
| `classes/objective_manager.php` | `mastery_threshold` | `DEFAULT_THRESHOLD` (0.85) | "no promotion to mastered" |
| `classes/objective_manager.php` | `mastery_window` | `DEFAULT_WINDOW` (8) | "ignore all attempts" |
| `classes/objective_manager.php` | `mastery_decay_half_life_days` | `DEFAULT_DECAY_HALF_LIFE_DAYS` (30) | "decay disabled" |
| `classes/attachment_manager.php` | `attachment_max_size_mb` | `DEFAULT_MAX_MB` (10) | "no attachment upload" |
| `classes/extractors/scorm_extractor.php` | `rag_scorm_max_mb` | `DEFAULT_MAX_MB` (10) | "skip SCORM in RAG" |

## Total ?: bugs fixed in the v5.3.31-v5.3.34 sweep

**18 distinct config keys** silently ignored when set to literal `0`, across ~15 files. Same root cause every time: PHP's `(int)"0"` is falsy, so `?: <default>` falls through and silently re-applies the default. The static-analysis test in `cron_tasks_test.php` now catches numeric, float, and constant defaults — both `self::CONST` and `OtherClass::CONST` — within `(int)` and `(float)` casts.

## Footnote: the mastery threshold tuning observation

Writing `test_compute_mastery_promotes_to_mastered_after_enough_correct_attempts` surfaced a tuning observation worth flagging: with the default constants (`PRIOR_ALPHA=2.0`, `PRIOR_BETA=2.0`, `DEFAULT_DECAY=0.9`, `DEFAULT_WINDOW=8`, `DEFAULT_THRESHOLD=0.85`), the mathematically-maximum achievable mastery score is approximately **0.79** — even for 8 perfectly-correct attempts at full weight. **No learner can ever reach `mastered` status with the default settings.** The test had to lower the threshold to 0.7 to assert the promotion path actually works.

This is not in scope for v5.3.34; documenting it here so it's visible. The fix is one of: lower `DEFAULT_THRESHOLD` to ~0.75; weaken the Bayesian prior (`PRIOR_ALPHA + PRIOR_BETA = 4` is heavy); or expand `DEFAULT_WINDOW`. Worth a separate conversation — the production mastery feature may have been tuned to be unreachable without anyone noticing.
